Introduction and Requirements

This program can edit save data from Final Fantasy IX. Made by the almighty Gjørulv, It can also work as a simplified PSX memory card editor, and, as of version 0.6b also supports formats for the 2016 re-release (for steam/android/ios ports). !The 2016 re-release is only tested on the steam version!.
I keep Memoria in the public domain, but I haven't published the source anywhere yet. Just kidding, it's HERE!

Memoria is for Windows. To run it, Microsoft .Net framework version 2.0 is required. You probably have it, unless you have a old computer and never update. If Memoria opens and runs the framework is installed. If you do not have it, you can download .NET HERE. Any version will do, as long as it's at least 2.0.

For Linux/OSx users: I suspect it's compatible with Wine, but I haven't tested it.

Questions beyond what is explained here can be asked by sending an e-mail to: gjoerulv@hotmail.com, or post in the forum.

Memory Card Handling

Memoria can open various types of PSX memory card files and single saves.

Memory Cards:

  • ePSXe/PSEmu Pro (*.mcr)
  • DexDrive (*.gme)
  • pSX/AdriPSX (*.bin)
  • Bleem! (*.mcd)
  • VGS (*.vgs, *.mem)
  • PSXGameEdit (*.mc)
  • WinPSM (*.ps)
  • Smart Link (*.psm)
  • MCExplorer (*.mci)
  • DataDeck (*.dff)

Single Saves:

  • XP, AR, GS, Caetla (*.psx)
  • Datel (*.pda, *.mcx)
  • Smartlink (*.mcb)
  • Memory Juggler (*.ps1)
  • PSXGameEdit (*.mcs)
  • Memoria Template (*.mmrb)
  • Raw Save

PS3 virtual saves (*.psv) and PSX PSP memory cards (*.vmp) can also be worked on, however, these files contains checksums that will be corrupted if anything is changed within the file. There are workarounds. Google it! I'm too lazy to explain right now. :P The "Save As..." feature works well if you wish to convert one type to another.

Once a card is opened, depending on the data, a grid will be filled based on FF9 previews. One preview pr. row, obviously. There are several columns in this view, most self-explanatory (you got this to edit FF9 saves, right?), but some may need some explanation.

Block: A PSX memory card contains 15 blocks, or frames with save data. 1 block equals 1 slot in byte length, but slot number 1, as seen in-game, may not be stored in block number 1. It depends on many things. What data you wish to edit is based on block/frame number, not slot number.

Slot: This the actual slot number as it appears in-game.

Region: Basically PAL or NTSC. It's content is based the region code for the save data. Because the game ID is different for many countries, I made a similar amount of region options. Regions can be cycled through by clicking "Cycle Regions" in the toolbox above the grid, or simply selecting a country. You may notice that the game time changes when changing between PAL and NTSC. This is because they don't have the same frame rate.

Some primitive memory card editing is possible: Delete blocks, copying/cut/paste blocks and formatting cards. More than this is not possible as far as standard memory card editing is concerned. Everything else requires some FF9 data, but it's possible to view product and game code in other titles by clicking "Block Info...", or Ctrl+I. Empty blocks or blocks that contains data from another title will not be editable.

It is possible to import/export single save files (it is not possible to export PSV files). Importing/exporting data assumes single-block saves. Memoria cannot import/export/copy/paste multi-block saves. Deleting multi-block saves are possible.

Editing a Save

By double-clicking a row with FF9 data, a new form with editable data representing the selected block will be opened. It is also possible to click "Edit..." (1st column) or hit spacebar to open this form.
On the top is a dropdown menu where you can quickly jump to other blocks containing FF9 data. It's also possible to switch block by holding Alt while pushing the directional keys. Alt+Left or Alt+UP loads previous block. Alt+Right or Alt+Down loads the next block.

Everything you edit will not take effect on the card unless you click OK or Apply at the bottom. Clicking Cancel removes all edits since the last time you clicked Apply, and closes the edit form. Clicking OK also closes the form, but everything edited will be remembered. Memoria will remember everything you edit, even when switching blocks. If the Apply button is grayed out (not enabled, or not clickable) everything is identical since you opened the edit form or since the last time you clicked Apply.

Once back at the main form you'll still need to save for anything to actually change on the file. The OK or Apply button is not equal to saving the changes in the file. By clicking "Undo" or Ctrl+Z anything you edited in the last session will be rolled back (if you actually did change something). Up to 4 actions can be undone/redone on the card; editing (a) save(s) counts as one action each time Apply or OK are clicked.

There are currently 5 main tabs in the Save Edit form: Characters, Party, Items, Cards, Hot & Cold and Misc. Each of these are explained more below.

You will notice there are export/import features for each category/tab, except Party and Misc. These are buttons with a green or red arrow. Red = export, green = import. Exporting/importing templates only effects the active category, and works cross files. In characters, there are 2 different export/import features: single character ("Misc." tab) or all characters at once ("All Characters" group box).

Characters

By clicking on a character portrait, the one you clicked is editable. Some characters contains data for two. It's possible to view other appearances by selecting Yes in the Display as... radio buttons. This is only for display purposes. As for the actual editable data it should be self-explanatory.

Total stats means base + item bonus. Base HP/MP are Max HP/MP without percentage bonuses from passive abilities.

Each status is one bit each; a higher number equals more statuses. 0 = no status, 255 = all statuses. And the bits are like this: First (1) is Petrification, second (2) is Venom, third (4) Virus, fourth (8) Silence, fifth (16) Darkness, sixth (32) Trouble, seventh (64) Zombie. It also seems the eight bit (128) has to be checked (any bit + 128, for instance: Virus (4) + 128 = 132) to get any status, but I may be wrong.

Pools have something to do with any +stats from gear every time a character levels up, but I'm not exactly sure how it works. It may also cause some unexpected results if these values are too high.

Actions like "Max All", or "Empty All" does not alter unknown values. These values are editable at own risk. Please report back if you figured out what they do, to: gjoerulv@hotmail.com

Party

There really isn't much to edit here.
Changing party members will update the preview data including leader name and level. Adding Beatrix to the party is possible as well. It's even possible to have an empty party, but I wouldn't recommend it.

Other than editing the members you can also change gil, successful thefts by Zidane, frogs captured by Quina and dragons slain. The number of dragons the party has killed only effects Freya's "Dragon's Crest" ability. 100 is enough. Also, frogs captured don't need to be more than 255 to max Quina's "Frog Drop".

It isn't possible to export/import any of these values as a standalone template.

Items

There are 3 different ways of editing items.

1. Directly: By clicking on the selected item's name, a drop-down box will appear. Now you can select which item the selected slot should hold. Same thing with quantity, but here it's not a drop-down box. You can type the number or push Up or Down on the keyboard to increase or decrease values. With item's name, when the drop-down box is visible, it's possible to jump to items by pushing the 1st letter (in the name) on the keyboard. Hitting enter, or simply leaving the controls, updates the values.
If you hit Delete when an item is selected, the selected slot will be emptied.

2. Selected Item: Here you can choose what item to edit. When you change it you'll notice selected item change as well in the item view. When clicking "Fill" on an empty slot it'll be 99 Potions.

3. All Items: The one feature to point out is the "macro" function which alters item's quantities. Select/type a number, choose function in the drop-down box (add, subtract, etc) and hit "!". Depending on the previous controls, this will affect all items. One example: You typed 1,50 in the box, selected "Multiply", then hit "!". Now all item's quantity will be multiplied by 1,5.

You can export/import item templates.

Cards

On the left are the editable controls. On the right are the cards. Clicking on a card makes it active.

Editing all cards contains similar "macro" actions as editing all items does, but there are more properties to handle on cards. Each card has attack, attack type, defence and magic defence values. All these, except attack type, has a max value of 255. If you divide this value by 16 you'll find the card's attribute value as it's displayed in-game.

When editing all cards, the red X means delete all cards. The green V means master all cards (Even empty cards; does not change card type). Clicking "All Types" gives you all type of cards from card 1-100 (by type it means Goblin, Fang, etc.). There is also a button ("Random") at the bottom right which randomizes the arrows of all cards.

To change the arrows on a single card, click on the selected card's picture. For instance, to toggle the upper left arrow, click on the upper left on the picture.

Miscellaneous and Unknowns

Not much to say here. In my laziness I've kept disc nr zero based, which means the actual disc nr is one more than what's displayed here.

Concerning unknowns: All unknown bytes can be editing in a hex-editor. Click on the range of bytes you wish to edit, and hex away. Of course, as in unknown character bytes, I would like to know what they represent. if you happen to find out, please, let me know: gjoerulv@hotmail.com
For re-release 2016 version, all data is editable in hex.

Wins, Losses and Draws refer to card games.

The "?" in jump ropes are unknown, but it's value seems always to be 0.
